你现在调用的是【Backlog 拆解】命令。

请将以下 PRD / 需求拆解为可直接执行的任务清单。

## 拆解层级

```
Epic（史诗）
  └── Story（用户故事）
        └── Task（任务）
```

### Epic
- 大型功能模块
- 通常需要多个迭代完成
- 例如："用户系统"、"商品管理"

### Story
- 可独立交付的功能单元
- 端到端用户价值
- 可在一个迭代内完成
- 格式：As a [角色], I want [功能], so that [价值]
- 例如："作为买家，我希望能搜索商品，以便找到我想要的商品"

### Task
- 具体执行任务
- 可直接分配给工程师
- 时间估算：0.5-3天
- 例如："设计商品搜索接口"、"实现搜索功能前端"

## 拆解原则

### Story 要求
- **独立**: 可独立开发和交付
- **可估算**: 可估算工作量
- **可测试**: 有明确验收标准
- **有价值**: 对用户有价值

### Task 要求
- **具体**: 明确做什么
- **可执行**: 技术人员可直接执行
- **可验收**: 有明确完成标准
- **合理**: 工作量合理（0.5-3天）

## 输出格式

```markdown
# Backlog 拆解

## Epic: [Epic名称]

### Story: [Story名称]
**描述**: As a [角色], I want [功能], so that [价值]

**验收标准**:
- [ ] AC-1: ...
- [ ] AC-2: ...

**任务拆解**:
- [ ] Task 1: [任务描述] (2天) - [负责人]
- [ ] Task 2: [任务描述] (1天) - [负责人]
- [ ] Task 3: [任务描述] (0.5天) - [负责人]

**估算**: 总计 3.5 天

**依赖**:
- 前置依赖：...
- 被依赖：...

---

### Story: [Story名称]
...
```

## 完整性检查

每个 Epic 应包含：
- [ ] 产品/设计任务
- [ ] 后端开发任务
- [ ] 前端开发任务
- [ ] 测试任务
- [ ] 上线/部署任务

## 任务类型

- **Product**: PRD、UX设计、UI设计
- **Backend**: 接口设计、数据库设计、后端开发
- **Frontend**: 组件开发、页面开发、状态管理
- **Test**: 单元测试、集成测试、E2E测试
- **DevOps**: 部署、配置、监控

## 输入格式

```
PRD内容：[PRD文本或要点]

或

需求列表：
1. [需求1]
2. [需求2]

约束条件（可选）：
- 团队配置：[例如：1后端 + 1前端]
- 时间限制：[例如：2周]
- 技术栈：[例如：React + Node.js]
```

## 估算建议

- **粗略估算**: Story级别（人天）
- **详细估算**: Task级别（人时）
- **Buffer**: 预留20%缓冲
- **风险**: 复杂任务增加50%缓冲

请开始拆解 Backlog。
